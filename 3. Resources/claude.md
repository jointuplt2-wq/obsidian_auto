# 3. Resources

## 목적과 역할
현재 프로젝트/영역에 묶이지 않은 **관심 주제별 참고 자료**의 보관소입니다.
"언젠가 쓸 수도 있는" 외부 정보의 도서관.

## 어떤 노트가 들어가는가
- 주제별 자료 모음 (예: "프로그래밍", "디자인 레퍼런스", "여행")
- 외부 링크 모음, 스니펫, 가이드, 치트시트
- 아직 소화되지 않은 수집물 (소화되면 Zettelkasten으로 승격)

## 작업 방식과 규칙
- frontmatter:
  ```yaml
  type: resource
  status: active | archived
  created: YYYY-MM-DD
  updated: YYYY-MM-DD
  ```
- **Resources ≠ Zettelkasten**: 여기는 "외부 자료 보관", Zettelkasten은 "내가 소화한 지식".
  읽고 내 생각으로 재구성하면 `5. Zettelkasten/`로 옮긴다.
- 3개월 이상 손대지 않은 자료는 `4. Archive/` 이동을 검토한다.
