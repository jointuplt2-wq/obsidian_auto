# 00. Inbox

## 목적과 역할
모든 아이디어의 **임시 착륙장**. 마찰 없이 빠르게 포착하는 곳이며, 영구 보관소가 아닙니다.

## 어떤 노트가 들어가는가
- 떠오른 생각, 짧은 메모, 처리 전 단상
- 아직 출처/분류/연결이 정리되지 않은 모든 것

## 작업 방식과 규칙
- **속도 우선**: 형식·태그·링크에 신경 쓰지 말고 일단 적는다.
- frontmatter는 최소만:
  ```yaml
  type: inbox
  created: YYYY-MM-DD
  ```
- **주 1회 비우기**: 각 메모를 처리한다 →
  - 외부 자료 정리 → `10. Literature/`
  - 내 통찰로 재구성 → `20. Permanent/`
  - 실행 작업 → `1. Projects/` 또는 `2. Areas/`
  - 불필요 → 삭제
- Inbox에 노트가 쌓여 있으면 그건 "처리 대기" 신호다. 영구 거주 금지.
