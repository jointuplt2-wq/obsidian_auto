# 운영 가이드

## PARA 분류 기준

### 1. Project
- 끝이 명확함
- 실행도가 가장 높음
- ex) 자격증 합격하기, 시험 공부, 블로그 글 작성, 제품 런칭, 전자책 출간, 이사하기

### 2. Area
- 끝이 없음. 진행 중
- 나의 책임이나 역할이 있음
- ex) 운동, 건강, 경제 계획, 직업, 가족

### 3. Resource
- 내가 관심 있는 분야
- 내 책임은 없음
- ex) 음악, 커피, 미술, 역사, 뇌과학, 마케팅, 글쓰기

### 4. Archive
- 실행도가 0임
- 종료된 프로젝트, 중단된 프로젝트
- 책임이 사라진 영역, 관심이 사라진 자원

## Hub 파일 운영 규칙

### Hub란
각 폴더 최상단에 고정되는 **폴더 전용 색인 노트(MOC)**. 파일명은 `00_<폴더>_Hub.md`
(`00_` 접두어로 파일 탐색기 맨 위 고정). 전역 요약은 [[00 Dashboard]],
폴더별 작업대는 Hub — 역할을 분리한다.

### 대상 (4개)
| 파일 | 폴더 | 역할 |
|---|---|---|
| `00_Projects_Hub.md` | 1. Projects | 진행/마감임박/완료 대기 프로젝트 색인 |
| `00_Areas_Hub.md`    | 2. Areas    | 활성 영역과 연결 프로젝트 현황 |
| `00_Resources_Hub.md`| 3. Resources| 주제별 자료 색인 |
| `00_ZK_Hub.md`       | 5. Zettelkasten | Inbox→Literature→Permanent 흐름, 고아 노트 |

### 필수 구성
1. **Frontmatter**: `type: hub` (Dataview 집계에서 제외되는 타입)
2. **폴더 소개**: 역할 1~2줄 + `[[운영-가이드]]` 링크
3. **Dataview 자동 색인**: 상태별·최근순 목록 (핵심)
4. **정리 체크리스트**: 해당 폴더 CLAUDE.md의 주기적 정리 항목 연결

### 원칙
- 내용은 **참조만** — 노트 본문을 Hub에 복사하지 않고 `[[링크]]`로 연결
- 수동 목록보다 **Dataview 우선** — 파일 추가 시 자동 갱신되게
- Hub는 폴더당 1개, `type: hub`는 Hub 전용 (일반 노트에 쓰지 않음)
- PARA 폴더는 `created`/`updated` 대신 OS 메타(`file.mtime`) 사용

> 참고: **project-hub 템플릿**은 이것과 다르다. 그건 *하위 프로젝트가 여럿 딸린
> 대형 프로젝트 1건*을 위한 허브이며, 폴더 색인용 Hub가 아니다.
